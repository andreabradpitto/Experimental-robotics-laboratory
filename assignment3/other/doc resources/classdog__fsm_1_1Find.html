Find state execution: in this state, the robotic dog looks for the goal ball, determined in the Play state.

In order to do so, this state relies on the explore_lite algorithm; a service client sends a flag to the server, which corresponds to the signal the server itself is waiting it order to run the above mentioned algorithm. The code of the algorithm, which is contained inside the Explore class (see explore.cpp), has been adjusted with the inclusion of a method able to handle requests coming from this node. The exploration algorithm keeps running until dog_vision.py stops its execution, i.e. a new ball has been found. If the new ball is indeed the goal one, the robot eventually gets back to the Play state. The human will then immediately call the dog back to its position, and another Play state cycle can begin
